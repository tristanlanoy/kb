# Getting Started

This section helps you get started with organizing your knowledge base.

## Adding New Pages

To add a new page to your knowledge base:

1. Create a new Markdown file in the `docs/` directory
2. Write your content using Markdown syntax
3. Optionally, organize files into subdirectories for better structure
4. Update `mkdocs.yml` if you want custom navigation

## Markdown Features

This knowledge base supports many Markdown extensions:

### Code Blocks

    print("Hello, World!")

## Organizing Content

You can organize your content by:

- Creating folders within `docs/` for different topics
- Using descriptive file names
- Adding a navigation structure in `mkdocs.yml`
